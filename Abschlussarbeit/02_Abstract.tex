\chapter*{Abstract}
This thesis addresses the implementation of a fiducial-based localization system for a \gls{uav} in indoor environments.
Indoors, no \gls{gnss} is available, so the absolute position reference usually relied upon by the flight controller is absent.
The aim of this work is to localize a \gls{uav} in a GPS-denied indoor environment and to feed this position estimate into the flight controller as an external reference.
The system is based on AprilTag markers of the \texttt{tag36h11} family, whose poses are stored in a map created beforehand.
The map is generated in a three-stage offline process using \gls{slam}.
During flight, the companion computer continuously determines the pose of the aircraft from the marker detections of two cameras and transmits it via \gls{uxrce}-DDS to the \gls{px4} flight controller, where it is processed in the \gls{ekf2} as an external position reference.
The system is built entirely from open-source software and off-the-shelf hardware.
The thesis describes the selection of the mapping and marker methodology, the camera calibration, the map creation, and the in-flight pose estimation together with its integration into the flight controller.

\chapter*{Kurzfassung}
\addcontentsline{toc}{chapter}{Kurzfassung}
\noindent
Die vorliegende Arbeit befasst sich mit der Implementierung eines fiducialbasierten Lokalisierungssystems für ein Unmanned Aerial Vehicle (UAV) im Innenraum.
In Innenräumen steht kein Global Navigation Satellite System (GNSS) zur Verfügung, sodass die sonst übliche absolute Positionsreferenz für die Flugsteuerung entfällt.
Ziel der Arbeit ist es, ein UAV in einer GPS-freien Innenraumumgebung zu lokalisieren und diese Positionsschätzung als externe Referenz in die Flugsteuerung einzuspeisen.
Als Grundlage dienen AprilTag-Marker der Familie \texttt{tag36h11}, deren Lage in einer vorab erstellten Karte hinterlegt ist.
Die Karte wird in einem dreistufigen Offline-Verfahren mit Simultaneous Localization and Mapping (SLAM) erstellt.
Im Flug bestimmt der Companion-Computer aus den Markerdetektionen zweier Kameras fortlaufend die Pose des Fluggeräts und übergibt sie über micro eXtremely Resource Constrained Environments Data Distribution Service (uXRCE-DDS) an den Pixhawk (PX4)-Flugcontroller, wo sie im Extended Kalman Filter 2 (EKF2) als externe Positionsreferenz verarbeitet wird.
Das System ist vollständig aus quelloffener Software und handelsüblicher Hardware aufgebaut.
Die Arbeit beschreibt die Auswahl der Kartierungs- und Markermethodik, die Kalibrierung der Kameras, die Kartenerstellung sowie die im Flug ablaufende Positionsbestimmung und deren Anbindung an die Flugsteuerung.
